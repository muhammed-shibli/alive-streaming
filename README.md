# Alive Streaming

A production-ready Flutter live-streaming app skeleton with **Firebase Google
Authentication**, a pixel-true Splash → Login → Home flow, and a clean
**MVVM + Clean Architecture** structure that is REST-API-ready (Dio).

---

## 1. Features

| Screen   | Purpose                                                                                 |
|----------|-----------------------------------------------------------------------------------------|
| Splash   | Animated logo + branded gradient. Bootstraps auth state and routes accordingly.         |
| Login    | Email/Password UI (visual only) + **Continue with Google** (real Firebase sign-in).     |
| Home     | Custom category tabs, scrollable region chips, live stream grid, custom bottom nav bar. |

- Stays logged in across launches (SharedPreferences + Firebase session).
- Cannot enter Home without successful Google sign-in.
- Responsive UI (scales for all phone widths via `R.of(context)` helper).
- Smooth animations on splash, button presses, follow chip, tab underline.

## 2. Tech Stack

| Concern         | Package / Pattern                                |
|-----------------|--------------------------------------------------|
| Architecture    | Clean Architecture (`core` / `data` / `presentation`) + MVVM |
| State           | `provider` (`ChangeNotifier` ViewModels)         |

## 3. Project Structure

```
lib/
├── main.dart                     # MultiProvider + MaterialApp + theme + routes
├── firebase_options.dart         # Generated by FlutterFire CLI
├── core/
│   ├── api/api_client.dart       # Dio singleton + interceptors
│   ├── constants/                # Colors, strings, text styles, asset paths
│   ├── routes/app_routes.dart    # Named routes
│   ├── services/
│   │   ├── auth_service.dart     # FirebaseAuth + GoogleSignIn wrapper
│   │   └── storage_service.dart  # SharedPreferences wrapper
│   └── utils/responsive.dart     # Responsive sizing helper
├── data/
│   ├── models/                   # UserModel, StreamModel (with fromJson/toJson)
│   └── repositories/             # AuthRepository, StreamRepository
└── presentation/
    ├── viewmodels/               # AuthViewModel, HomeViewModel
    ├── screens/
    │   ├── splash/               # SplashScreen
    │   ├── login/                # LoginScreen
    │   └── home/                 # HomeScreen + widgets/
    └── widgets/                  # PrimaryButton, CustomTextField, SocialButton
```

### Layer responsibilities
- **core** – pure infrastructure, no UI, no business state.
- **data** – models + repositories (single source of truth for app data).
- **presentation** – ViewModels (`ChangeNotifier`) + Widgets/Screens. Views
  never talk directly to services; they go through repositories via VMs.

## 5. Firebase Google Auth – setup checklist

1. **SHA-1 / SHA-256 fingerprints** for *debug* and *release* keystores must
   be added to the Firebase Android app
   (Project settings → Your apps → Add fingerprint), otherwise the Google
   sign-in dialog will close immediately with a `12500` error.
2. `google-services.json` must live in `android/app/`. *(Already present.)*
3. In Firebase Console → Authentication → Sign-in method, enable **Google**.

## 6. State Management (Provider)

Two `ChangeNotifier` ViewModels are registered at the app root in `main.dart`:

```dart
MultiProvider(
  providers: [
    ChangeNotifierProvider(create: (_) => AuthViewModel()),
    ChangeNotifierProvider(create: (_) => HomeViewModel()),
  ],
  child: MaterialApp(...),
);
```

- `AuthViewModel`  → bootstrap, signInWithGoogle, signOut, exposes
  `AuthStatus { unknown, loading, authenticated, unauthenticated }`.
- `HomeViewModel`  → active tab, region filter, bottom nav index, streams list,
  follow toggle. Loads via `StreamRepository`.

Views read with `context.watch<T>()` and call methods through
`context.read<T>()`.

## 7. REST API Readiness

`core/api/api_client.dart` exposes a Dio singleton (`ApiClient.instance.dio`).
To wire a real backend:

1. Update `ApiClient.baseUrl`.
2. Add a token interceptor in `ApiClient._internal()`.
3. Replace the mock in `StreamRepository.fetchStreams()` with:
   ```dart
   final res = await ApiClient.instance.get<List>('/streams', query: {'region': region});
   return res.data!.map((j) => StreamModel.fromJson(j)).toList();
   ```

The rest of the app does not need to change — models already implement
`fromJson` / `toJson`.
